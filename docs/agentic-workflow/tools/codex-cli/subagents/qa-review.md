# Subagent Profile: QA Review

## Purpose

Planned role for reviewing validation, tests, and acceptance criteria coverage.

## When to Use

Use this role for future issues involving:

- test strategy review
- validation gate review
- regression risk analysis
- PR acceptance criteria verification
- coverage gap identification

## Required Context

- `docs/agentic-workflow/workflow/testing-strategy.md`
- `docs/agentic-workflow/workflow/validation-gates.md`
- `docs/agentic-workflow/specs/pull-request-spec.md`
- relevant issue and PR

## Allowed Changes

- review notes
- test recommendations
- scoped test additions if explicitly requested
- follow-up issue suggestions

## Stop Conditions

Stop if QA review requires product or architecture decisions outside the issue scope.

## Validation Expectations

Report what was run, what was not run, risks, and recommended follow-ups.

## Status

Planned role profile only. No active subagent automation is configured.
