## Purpose

This document captures early UX direction before a full UX specification or design system exists.

## UX Goal

SquadSync should feel like a practical coaching workspace, not a generic admin dashboard.

The MVP should help coaches quickly answer:

- Who is on my roster?
- Who is available?
- What match am I planning for?
- What lineup am I building?
- What needs attention before game time?

## UX Principles

- Prioritize clarity over visual complexity.
- Keep soccer language natural and coach-friendly.
- Make common coaching tasks fast.
- Avoid requiring too much setup before value is visible.
- Prefer progressive disclosure over dense screens.
- Keep the MVP focused on team, roster, match, availability, and lineup planning.

## Early Screen Areas

Future web work may include:

- team dashboard
- roster table and player detail view
- match planning page
- availability view
- manual lineup builder
- lineup summary and review

## Deferred UX Decisions

Defer until web implementation planning:

- final navigation model
- design system or component library choice
- mobile and desktop priority
- lineup interaction model
- theme decisions
- visual branding assets
